Detailed Description

A* Pathfinding implementation for 2D and 3D grids.

Uses a binary min-heap as the open list priority queue for O(log n) insert/extract operations, and a flat array for O(1) closed-set membership checks.

Based on:

  • GameDev.net Article 2003: "A* Pathfinding for Beginners"
  • Red Blob Games: "Implementation of A*"

◆ ASTAR_BOX

struct ASTAR_BOX

Search window: the bounding box of (start, goal) expanded by ASTAR_SEARCH_MARGIN and clamped to the grid.

The per-call ASTAR_CELL working set is allocated for this window only, NOT the whole grid, so a short path on a huge map costs a few KB instead of width*height*depth cells. The grid's own walkable[] / cost[] planes stay full-size and are still indexed by the absolute grid_index().
