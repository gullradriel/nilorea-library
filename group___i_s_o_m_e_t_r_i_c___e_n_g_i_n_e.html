Applied as additional multiply on top of height tinting. Default: (1.0, 1.0, 1.0) = no change. Set by client lighting system each frame.

ISO_DRAW_ENTRY * cached_draw_order Cached draw_order.

The qsort+build in iso_map_build_draw_order is expensive (1024+ entries x 25 chunks per frame = ~3 ms in profiled bench). Cache the result on the map and only rebuild when segments or heights change. Setters that mutate either field (iso_map_set_segments is the only public one) set draw_order_dirty = 1. iso_map_draw rebuilds into cached_draw_order when dirty, then reuses across frames. ◆ IsoPrimitiveColorFilter

typedef ALLEGRO_COLOR(* IsoPrimitiveColorFilter) (ALLEGRO_COLOR c)

Optional design-RGB to display-RGB filter applied to every primitive color (filled polygons, lines) emitted by iso_map_draw.

Used by the client to opt into sRGB-correct rendering: when an sRGB FBO is bound and GL_FRAMEBUFFER_SRGB is enabled, RGB literals authored in sRGB space must be hand-converted to linear before being written, otherwise the GPU's linear-to-sRGB encode at FBO store time would double-encode them. Default NULL = identity passthrough; non-sRGB consumers (editor, headless tests) pay nothing. Set via iso_set_primitive_color_filter.

◆ n_iso_pass1_us

double n_iso_pass1_us
extern

Per-pass timing accumulators populated by iso_map_draw.

Each iso_map_draw() call adds its per-pass elapsed time (microseconds) to these globals. The host application is responsible for resetting them to 0.0 at the start of every measurement window (typically frame start), then reading them after the world render block to see how the cost split across the three internal passes:

  • n_iso_pass1_us: Pass 1: ground tiles (object-free loop)
  • n_iso_pass2_us: Pass 2: elevated segments + interleaved object callbacks (includes time spent inside N_ISO_OBJECT.draw callbacks; subtract any host-side callback timing to isolate the library's segment work)
  • n_iso_pass3_us: Pass 3: occlusion ghost overlay

Always populated. The overhead is six al_get_time() calls per iso_map_draw invocation: empirically < 1 us total, negligible vs the milliseconds-scale work the function does. No runtime flag needed.
